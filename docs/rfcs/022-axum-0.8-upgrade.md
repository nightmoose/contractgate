## Summary

Promote Dependabot #72 from "auto-merge blocked, treat as planned task" to a
deliberate, RFC-tracked upgrade. The PR bumps `axum` from 0.7 to 0.8 and
sweeps a cohort of downstream crates that broke API along with it. None of
this is auto-mergeable; route syntax changes alone touch ~25 call sites.

The validation engine (sub-15ms p99) is untouched — this is a transport-layer
upgrade only.

---

## Goals

1. Land axum 0.8 cleanly with zero behaviour change observable to API clients.
2. Bump cohort crates in lock-step (sha2, hmac, dashmap, tower, axum-test) so
   the lockfile stops drifting against Dependabot weekly.
3. Maintain or improve compile time; do not introduce new system deps.
4. All CI green: `cargo check`, `cargo test`, compose-smoke, compose-demo-smoke.
5. Validation engine p99 stays ≤ 15 ms (verify via existing bench).

## Non-goals

- sqlx upgrade (already at 0.8 — memory was stale; no action needed).
- tower-http 0.5 → 0.6 (separate RFC if/when Dependabot raises it).
- axum 0.8's new typed-state ergonomics — refactor is a follow-up, not a
  blocker.
- Replacing `axum-test 20` with reqwest-based integration tests.

## Breaking changes (axum 0.8)

### 1. Path parameter syntax — `/:param` → `/{param}`

**Impact: every route with a path param.** axum 0.8 uses `matchit` 0.8 syntax
to align with OpenAPI / RFC 6570 conventions. Old `:param` no longer parses.

Affected routes (grepped from `src/main.rs`):

```
/contracts/:id
/contracts/:id/name-history
/contracts/:id/versions
/contracts/:id/versions/:version
/contracts/:id/versions/latest-stable
/contracts/:id/versions/:version/promote
/contracts/:id/versions/:version/deprecate
/contracts/:id/versions/:version/export
/contracts/:id/versions/:version/approve-import
/contracts/:id/versions/:version/odcs-conformance
/contracts/:id/quarantine/replay
/contracts/:id/quarantine/:quar_id/replay-history
/ingest/:raw_id
/ingest/:contract_id/stats
/v1/ingest/:contract_id
```

All become `{id}`, `{version}`, `{contract_id}`, `{quar_id}`, `{raw_id}`.
Handler signatures (`Path<String>`, `Path<(String, String)>`) are unchanged.

OpenAPI spec (`v1_ingest::openapi_handler`) already uses `{contract_id}` —
no change required there.

### 2. Trailing-slash redirect removed

axum 0.8 no longer auto-redirects `/foo/` → `/foo`. We do not depend on this
behaviour (verified by grep — no client-facing docs mention trailing slashes),
but the compose-smoke suite must be re-checked after the bump.

### 3. `Handler` / `IntoResponse` trait tightening

`#[axum::debug_handler]` is stricter about `Send + 'static` on captured state.
Likely no-op for our codebase since all state is `Arc<AppState>`, but expect
1–2 trivial annotation fixes.

### 4. `axum-test` 20 → 21 (or current)

Aligns with axum 0.8. Test ergonomics are unchanged for our usage; integration
tests in `tests/` should compile after a version bump and route-syntax sweep.

## Migration plan

Single PR, single commit per logical step so review is easy:

1. **Branch** from `main`: `nightly-maintenance-2026-05-02-axum-upgrade`.
2. **Bump `Cargo.toml`** axum + cohort crates to target versions; run
   `cargo update -p axum -p tower -p sha2 -p hmac -p dashmap -p axum-test`.
3. **Route sweep** — `src/main.rs` only. Convert all `:param` to `{param}`.
   Single commit, mechanical change. Verify via `cargo check`.
4. **Handler / state fixups** — fix any `debug_handler` complaints. Expected
   small (1–2 sites).
5. **PII transforms** — verify `sha2 0.11` import paths in `src/transform.rs`;
   adjust if `Digest` re-export moved.
6. **Tests** — bump `axum-test`, fix any test-only API drift.
7. **CI** — confirm `cargo check`, `cargo test`, compose-smoke, compose-demo-
   smoke, security job all green.
8. **Bench** — run validation-engine bench; record p99 in PR description.
   Block merge if p99 > 15 ms or regresses > 10 % from baseline.
9. **Squash-merge** to `main`; close Dependabot PR #72 with
   "superseded by #<PR>".
10. **Update memory** — `project_nightly_2026-05-02.md` to drop the "axum
    upgrade pending" note.

Estimated effort: 2–4 hours including bench.

## Test plan

- `cargo test` — full unit + integration suite must pass.
- `cargo audit` — no new advisories.

## Risks

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Missed `:param` route → 404 in prod | Low | grep is exhaustive; CI smoke covers all top-level paths. |
| Hidden tower 0.5 API break in middleware stack | Med | `require_api_key` and rate-limit layers are thin; isolate fix in step 4. |
| sha2 0.11 import drift breaks PII hashing | Low | `transform.rs` is small, well-tested. |
| Validation p99 regresses | Very low | Pure transport upgrade; no validation-path changes. Bench gate catches it. |
| axum-test bump cascades into integration test rewrites | Med | Time-box at 1 hour; if larger, defer test bump and pin axum-test 20 against axum 0.8 if compatible. |

## Rollback

Revert merge commit. Lockfile restoration is automatic. Dependabot will
re-raise within a week.
