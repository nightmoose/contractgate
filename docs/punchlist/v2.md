# Punchlist v2 — Pre-Customer Trim

**Created:** 2026-04-27 on branch `Punchlist_002`.
**Replaces:** the broader v1 in [README.md](README.md) (chunks 2–8).
**Lens:** no customers yet. Build only what supports first pilots and patent-core demos. Defer everything that needs scale signal to be designed correctly.

## Build now (5 chunks, 5 conversations)

| # | Chunk | RFC | Why now |
|---|---|---|---|
| 1 | CLI core + reference GH Actions workflow | [RFC-014](../rfcs/014-cli-core.md) | Every pilot needs `push`/`pull`/`validate`. |
| 2 | Breaking-change demo arc | [RFC-015](../rfcs/015-breaking-change-demo.md) | Patent-core sell. Taxonomy → severity → migration suggester → CLI diff → audit search. |
| 3 | Observability v1 (metrics only) | [RFC-016](../rfcs/016-observability-v1.md) | `/metrics` + Grafana JSON. Alerts deferred until first pilot has live traffic to design thresholds against. |
| 4 | Onboarding stack | [RFC-017](../rfcs/017-onboarding-stack.md) | Compose stack + 3 starter contracts + **demo seeder** (posts realistic events through `/ingest` so RFC-016 dashboards and RFC-015 audit search look real on a fresh boot). |
| 5 | TypeScript SDK | [RFC-018](../rfcs/018-typescript-sdk.md) | Broadest pilot reach. Single language client. |

## Deferred (revisit when first customer signal arrives)

| Item | Original RFC | Why defer |
|---|---|---|
| Multi-tenant isolation, signup, rate limits, billing | RFC-013 | No tenants, no plans. Will mis-design without real usage. |
| Helm chart, RBAC role editor, air-gap bundle | RFC-010 (most) | No on-prem deploys yet. |
| Java + Go SDKs | RFC-011 (most) | TypeScript covers the early-pilot spectrum. |
| Terraform provider, K8s Operator | RFC-013 tail | Massive lift, no users to consume them. |
| Template registry API, browser tab, ratings, submission, private namespaces | RFC-012 (most) | Three starter YAML files cover the onboarding need. Network effect needs a network. |
| Slack alerts, email digest, PagerDuty, generic webhooks, full rule engine | RFC-009 entire alerting stack | No live traffic yet — alerts would never fire on test data. Returns as follow-up RFC when first pilot has continuous flow. |
| Impact estimator, materialized view, `/contracts/:id/impact` | RFC-008 partial | No audit volume yet. Producer counts would be ≤1 every time. |
| Pre-commit hook framework, GitLab/Bitbucket templates | RFC-007 tail | GitHub workflow is the proof. |
| GitOps reconciliation loop | RFC-007 tail | Adds a server process; not needed for pilots. |
| Anomaly detection, real-time health dashboard | RFC-009 tail | Needs `/metrics` data flowing first. |
| LLM-backed migration / diff prose | RFC-006/-008 reserved slots | Rule-based ships demos; LLM is a feature add. |
