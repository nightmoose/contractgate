## 🔴 Critical

### ✅ 1. Compose smoke CI — `GET /contracts` 500 — RESOLVED (2026-05-02)
**Root cause:** `ops/postgres/initdb-wrapper.sh` committed as `100644` (not executable).
Docker's `/docker-entrypoint-initdb.d/` only runs `.sh` files with the executable bit set —
migrations never applied, `contracts` table didn't exist, every DB query returned
`ERROR: relation "contracts" does not exist` → 500.
**Fix:** `c29391d` set mode to `100755`. Verified in CI.

**Also fixed same session:** `tests/compose_demo_smoke.sh` jq matcher bug — the expression
`(.contracts // .)[]` throws a runtime error when the response is a bare JSON array (not
wrapped in `{contracts: [...]}`). Silently returned 0, causing false `contract not found`
failures. Fixed to `(if type == "array" then . else (.contracts // []) end)[]`.
